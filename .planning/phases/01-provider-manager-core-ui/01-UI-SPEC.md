## Action Bindings (Pre-populated from CONTEXT.md D-03, D-04, D-05)

## Input Dialogs

### Add Provider (Pre-populated from CONTEXT.md D-03)

| Property | Value |
|----------|-------|
| Trigger | Ctrl-A |
| Method | `vim.ui.input` |
| Prompt | `New provider name: ` |
| Validation | `^[a-z][a-z0-9_-]*$` (kebab/snake case) |
| On valid input | Open providers.lua for config entry |

**Error feedback:** `vim.notify("Provider name must be lowercase with dashes/underscores", vim.log.levels.ERROR)`

### Delete Provider (Pre-populated from CONTEXT.md D-04)

| Property | Value |
|----------|-------|
| Trigger | Ctrl-D on selected provider |
| Method | `vim.ui.input` |
| Prompt | `Delete provider {name}? (y/n): ` |
| On 'y' | Remove provider from registry, notify success |
| On 'n' or cancel | Cancel operation, no action |

**Success feedback:** `vim.notify("Deleted provider: {name}", vim.log.levels.INFO)`

### Edit Provider (Pre-populated from CONTEXT.md D-05)

| Property | Value |
|----------|-------|
| Trigger | Ctrl-E |
| Method | `vim.cmd("edit lua/ai/providers.lua")` |
| Jump behavior | Jump to line with selected provider's `M.register()` call |

**Rationale:** Provider config involves multiple fields (endpoint, model, static_models, timeout). Direct file editing is more practical than multi-field input dialogs.

## Display Format

### Provider List (from model_switch.lua pattern)

```lua
-- Display format: "{name}  —  {endpoint}  —  {model}"
local display = string.format("%s  —  %s  —  %s", name, endpoint, model)
```

### Model List (from model_switch.lua pattern, beautify_model_item)

```lua
-- Display format: "{model_id}  —  {context_length}  —  {created_date}"
local label, id = Util.beautify_model_item(m)
```

**Fallback:** When dynamic fetch fails, use `static_models` array directly with placeholder metadata:
```lua
local label = string.format("%s  —  unknown  —  unknown", model_id)
```

### Keymap Registration

```lua
-- In lua/ai/init.lua or lua/ai/provider_manager/init.lua
vim.keymap.set("n", "<leader>kp", function()
  require("ai.provider_manager").open()
end, { desc = "AI Provider Manager" })

vim.api.nvim_create_user_command("AIProviderManager", function()
  require("ai.provider_manager").open()
end, { desc = "Open AI Provider Manager panel" })
```

### File Structure (from RESEARCH.md)

Create `lua/ai/provider_manager/` directory:
- `init.lua` — Orchestrator, command/keymap registration
- `registry.lua` — CRUD operations, delegates to providers.lua/keys.lua
- `picker.lua` — FZF-lua picker with CRUD actions
- `validator.lua` — Input validation for provider names
