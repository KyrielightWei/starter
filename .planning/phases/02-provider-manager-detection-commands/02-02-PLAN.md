    **Key fix from reviews (HIGH - unanimous):** Replace `io.popen` with `vim.system()` (Neovim 0.10+). This was the #1 concern from all 3 reviewers — io.popen blocks the main thread.

    **Async HTTP implementation (Qwen suggestion):**
    ```lua
    -- Injectable for testability
    M._http_fn = M._http_fn or vim.system

    function M._do_request(endpoint, api_key, model, timeout_ms, callback)
      local url = endpoint .. "/v1/chat/completions"
      local body = vim.json.encode({
        model = model,
        messages = { { role = "user", content = "hi" } },
        max_tokens = 1,
      })

      M._http_fn({
        "curl", "-s", "-m", tostring(math.floor(timeout_ms / 1000)),
        "-X", "POST",
        "-H", "Content-Type: application/json",
        "-H", "Authorization: Bearer " .. api_key,
        "-d", body,
        url,
      }, { timeout = timeout_ms }, function(obj)
        callback(obj)
      end)
    end
    ```

    **Core flow for check_provider_model:**
    1. Check cache via `Cache.is_valid()` first, return cached if valid
    2. Get provider config from `Providers.get()` — return error if not found
    3. Get API key from `Keys.get_key()` — return error if missing
    4. Get base URL from `Keys.get_base_url()` — return error if missing
    5. **Endpoint compatibility check** (GLM HIGH): Validate endpoint contains `/v1/` or ends with `/compatible-mode` — if not, warn via vim.notify and attempt anyway
    6. Build HTTP request (use `M._http_fn`) with timeout
    7. Parse response: `json.error` → unavailable, `json.choices` → available
    8. **Sanitize error messages** (Kimi LOW): `error_msg:gsub("sk-[A-Za-z0-9]+", "[KEY_REDACTED]"):gsub("Bearer [^ ]+", "Bearer [REDACTED]")`
    9. Cache result if status is "available"
    10. Return/callback with `{status, response_time, error_msg}`

    **Sync wrapper:** Uses `vim.wait()` to block until async callback fires:
    ```lua
    function M.check_single(provider_name, model_id)
      local result = nil
      local done = false
      M.check_provider_model(provider_name, model_id, function(r)
        result = r
        done = true
      end)
      vim.wait(provider.timeout or 30000, function() return done end, 50, false)
      return result
    end
    ```

    **Async batch (check_all_providers):**
    - Use simple recursive `run_next()` queue (from Qwen MEDIUM suggestion, not io.popen + vim.loop semaphore)
    - Max 3 concurrent via active counter
    - Show progress via `vim.notify("检测中: N/total", vim.log.levels.INFO, { replace = true })` — the `{replace=true}` prevents notification coalescing (Kimi MEDIUM)
    - Call callback with full results array when all complete
    - **Partial batch fault tolerance** (GLM HIGH): If one check fails, continue others; no retry for now (keep scope manageable)

    **Error status taxonomy** (from review consensus):
    - `available` — HTTP 200, valid JSON with choices
    - `unavailable` — 4xx/5xx, auth failure, connection refused
    - `timeout` — Request exceeded timeout
    - `error` — Parse failure, 200 but unexpected shape, or network error

    Follow Neovim Lua conventions: `local M = {}` pattern, snake_case, pcall for unsafe ops, 2-space indent, 120 char width.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| API Key → detector.lua | API keys read from Keys module, injected into curl Authorization header |
| curl output → detector.lua | Untrusted JSON response parsed safely via pcall + json_decode |
| Cache file → cache.lua | Cache read via dofile in pcall, stored in user-controlled stdpath("state") |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | detector.lua curl | mitigate | API key passed only via Authorization header; curl stderr/output not logged; no sensitive data in vim.notify messages; error messages sanitized with gsub before display |
| T-02-02 | Tampering | cache.lua (inherited) | mitigate | `pcall(dofile)` — returns empty table on parse failure; cache in stdpath("state") not world-writable (per Plan 01) |
| T-02-03 | Denial of Service | check_all_providers | mitigate | Max 3 concurrent requests prevents API rate limit exhaustion; per-request timeout prevents hanging |
| T-02-04 | Spoofing | curl response parsing | mitigate | Strict validation: must have `json.choices` for "available" status; `json.error` → "unavailable"; any other parse → "error" |
</threat_model>
