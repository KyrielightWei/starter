Required exports:

1. **M.get_cached_status(provider, model)**:
   - Require cache.lua
   - Guard: if provider or model is nil/empty string, return "unchecked" immediately (addresses C-14)
   - Call `Cache.is_valid(provider, model)` — if false, return string "unchecked"
   - Call `Cache.get(provider, model)` — extract `.status` field, return it
   - Note: No "pending" state here — this returns only cache state. Pending state comes from the separate `get_cached_status_with_pending` function.

2. **M.get_cached_status_with_pending(provider, model)**:
   - Same as get_cached_status but returns an additional boolean: `(status, is_checking)`
   - `is_checking` = true if we know detection is currently in flight for this provider+model
   - For initial implementation: return `(get_cached_status(provider, model), false)` — the `is_checking` boolean is reserved for future cache-aware tracking when detector exposes in-flight status

3. **M.trigger_async_check(provider, model, on_complete)**:
   - Require detector.lua
   - Capture the provider + model pair at call time for stale guard
   - Call `Detector.check_provider_model(provider, model, vim.schedule_wrap(function(result) ... end))`
   - In callback FIRST: check if `ai.state.get()` still matches the captured provider+model. If user has switched away, return silently — discard stale result (addresses C-02, C-11)
   - If current state matches: invoke `on_complete(result)` (already wrapped via vim.schedule_wrap)
   - If on_complete is nil, just run the check (fire-and-forget, no callback)
   - Use `vim.schedule_wrap()` directly — it is the idiomatic Neovim pattern (simpler than `vim.schedule(function() ... end)` wrapper)

4. **M.check_all_batch(callback)**:
   - Require detector.lua
   - Call `Detector.check_all_providers(vim.schedule_wrap(function(results) callback(results) end))`
   - Passes through the batch results unchanged

Module structure:
```lua
local M = {}

local Cache = require("ai.provider_manager.cache")
local Detector = require("ai.provider_manager.detector")
local State = require("ai.state")

function M.get_cached_status(provider, model)
  if not provider or provider == "" or not model or model == "" then
    return "unchecked"
  end
  if not Cache.is_valid(provider, model) then
    return "unchecked"
  end
  local entry = Cache.get(provider, model)
  return entry and entry.status or "unchecked"
end

function M.get_cached_status_with_pending(provider, model)
  local status = M.get_cached_status(provider, model)
  return status, false  -- is_checking = false (reserved for future in-flight tracking)
end

function M.trigger_async_check(provider, model, on_complete)
  local captured_provider = provider
  local captured_model = model
  Detector.check_provider_model(provider, model, vim.schedule_wrap(function(result)
    -- Stale guard: check if user is still on this provider+model
    local current = State.get()
    if not current or current.provider ~= captured_provider or current.model ~= captured_model then
      return  -- Discard stale callback
    end
    if on_complete then
      on_complete(result)
    end
  end))
end

function M.check_all_batch(callback)
  Detector.check_all_providers(vim.schedule_wrap(function(results)
    if callback then
      callback(results)
    end
  end))
end

return M
```

```lua
-- PMGR-07: Auto-detect availability in background before callback
local Status = require("ai.provider_manager.status")
Status.trigger_async_check(provider, model, function(result)
  if result and result.status ~= "available" then
    local msg = string.format("[AI] %s / %s 状态: %s", provider, model, result.status or "unknown")
    if result.error_msg and result.status == "error" then
      msg = msg .. " — " .. result.error_msg
    elseif result.status == "unavailable" then
      msg = msg .. " — 模型可能不可用"
    elseif result.status == "timeout" then
      msg = msg .. " — 检测超时"
    end
    -- C-01/C-10: vim.notify wrapped in vim.schedule for async callback safety
    vim.schedule(function()
      vim.notify(msg, vim.log.levels.WARN, { title = "AI Provider", replace = true })
    end)
  end
end)

if callback then
  callback({
    provider = provider,
    model = model,
  })
end
```

Key constraints per locked decisions:
- D-01: Detection is async — callback fires immediately, NOT after detection
- D-03: On failure/unavailability, show warning via vim.notify — do NOT prevent the switch
- D-09: User retains control — no auto-revert, no blocking

The `vim.notify` is wrapped in `vim.schedule()` to prevent Neovim error E5560. The `{ title = "AI Provider" }` gives the notification a label for better replace behavior. `{ replace = true }` prevents notification buildup if user switches models rapidly.

Use local require inside the callback closure (lazy loaded, avoids circular dependencies with ai/init.lua). Also add State.set call AFTER the callback fires to ensure ai.state reflects the new selection (this may already be done by the caller — verify before duplicating).
