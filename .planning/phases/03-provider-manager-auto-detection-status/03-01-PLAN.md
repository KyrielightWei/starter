```lua
-- PMGR-07: Auto-detect availability in background before callback
local Status = require("ai.provider_manager.status")
Status.trigger_async_check(provider, model, function(result)
  if result and result.status ~= "available" then
    local msg = string.format("[AI] %s / %s 状态: %s", provider, model, result.status or "unknown")
    if result.error_msg and result.status == "error" then
      msg = msg .. " — " .. result.error_msg
    elseif result.status == "unavailable" then
      msg = msg .. " — 模型可能不可用"
    elseif result.status == "timeout" then
      msg = msg .. " — 检测超时"
    end
    vim.notify(msg, vim.log.levels.WARN, { replace = true })
  end
end)

if callback then
  callback({
    provider = provider,
    model = model,
  })
end
```

Key constraints per locked decisions:
- D-01: Detection is async — callback fires immediately, NOT after detection
- D-03: On failure/unavailability, show warning via vim.notify — do NOT prevent the switch
- D-09: User retains control — no auto-revert, no blocking

The `vim.notify` uses `{ replace = true }` to avoid notification buildup if user switches models rapidly.

No new imports at top of file — use local require inside the callback closure (lazy loaded, avoids circular dependencies with ai/init.lua).
