    `get_commit_list(base, head, opts)`:
    - If base and head are nil → run `git log --max-count=20 --format=... HEAD` (fallback recent commits)
    - Otherwise → run `git log --format=%H%n%h%n%s%n%cr%n%d base..head`
    - Parse output in groups of 5 lines per commit
    - Return array of `{sha, short_sha, subject, date, refs}`
    - refs = branch/tag decorations from %d, trimmed (D-04 format: branch_info)

    `get_unpushed()`:
    - Wrapper: `get_commit_list("origin/HEAD", "HEAD")`
    - Returns empty array if no unpushed commits (ahead=0) or git error

    `get_ahead_behind()`:
    - Run: `git rev-list --left-right --count origin/HEAD...HEAD`
    - Parse stdout: first number = behind, second = ahead
    - Return `{ahead = N, behind = M}`

    Error handling (D-16):
    - Use pcall around io.popen
    - If git exits non-zero → capture stderr/output and return as error info: `{error = true, output = "..."}` to display in notification
    - If not a git repo → return empty
    - If no remote named origin → return empty

    Follow Neovim Lua conventions: `local M = {}`, 2-space indent, pcall for unsafe ops.

          width = 0.6,    -- 60% width

          ["default"] = function(selected)
            -- Parse SHAs from selected lines (strip bracket formatting)
            local shas = {}
            for _, line in ipairs(selected) do
              local sha = line:match("^%[(%x+)%]")
              if sha then table.insert(shas, sha) end
            end
            if opts.on_select then opts.on_select(shas) end
          end,

            return entry  -- fzf-lua handles highlighting

          return ""
        end,
