  <action>
Create `lua/commit_picker/navigation.lua` with commit cycling state management. The module tracks:
- Full commit list (array of {sha, short_sha, subject, date, refs})
- Current position index (1-based, pointing to the commit currently shown in diffview)
- Current view mode: "single" (sha^..sha) or "range" (sha1..sha2)

Required exports:

1. `M.setup(opts)` — Accepts `{ Git, Selection, Diff }` modules. Called from init.lua.

2. `M.load_commits()` — Fetches commits via `Git.get_commits_for_mode()`. If no commits found, fall back to `Git.get_commit_list(nil, "HEAD", {count = 20})`. Stores commits in module-local `commit_list` variable and resets position to 1. Returns commit count.

3. `M.cycle_next()` — Advances position by 1. If at end of list, display notification "已是最后一条提交" and stay. Otherwise: get current commit's SHA, close diffview if open (via `pcall(vim.cmd, "DiffviewClose")`), then open new diff: `DiffviewOpenEnhanced sha^..sha`. Update selection to [current_sha]. Show notification "已导航到 [N/M]: [subject]". Return new position.

4. `M.cycle_prev()` — Moves position back by 1. If at position 1, display notification "已是第一条提交" and stay. Otherwise: same flow as cycle_next (close diffview → open new diff → update selection → notify).

5. `M.get_position()` — Returns `{ position = N, total = M, commit = { ... } }` for status indicator use.

6. `M.get_current_sha()` — Returns full SHA at current position, or nil if no commits loaded.

7. `M.is_loaded()` — Returns true if commit_list is populated.

IMPORTANT DESIGN NOTES:
- Use `pcall(vim.cmd, "DiffviewClose")` before opening new diff to avoid "diffview already open" error. The existing `DiffviewOpenEnhanced` in `lua/plugins/git.lua` already includes the guard.
- Use `Diff.close_if_open()` pattern from `lua/commit_picker/diff.lua` — but note that diff.lua only checks if already_open and returns. For cycling, we MUST close then reopen.
- Module-local state should NOT persist across Neovim sessions (in-memory only, like selection.lua).
- Use `vim.notify()` with appropriate log levels: INFO for progress, WARN for boundary warnings.
- Follow project conventions: `local M = {}`, `function M.xxx()`, `return M` at end.
- Use `vim.defer_fn()` only if needed for async operations — navigation should be synchronous since diffview handles the diff rendering.
- IMPORTANT: When closing diffview, use `pcall(vim.cmd, "DiffviewClose")` — diffview.nvim may not always be open when cycling is triggered (user might close it manually then use navigation keymap). pcall prevents errors.
