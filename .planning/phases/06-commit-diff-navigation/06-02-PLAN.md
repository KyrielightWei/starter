Implementation:

```lua
local M = {}

-- Returns status string like " [3/20]" or "" if no commits loaded
function M.get_status()
  local ok, Nav = pcall(require, "commit_picker.navigation")
  if not ok or not Nav.is_loaded() then
    return ""
  end
  local pos = Nav.get_position()
  if not pos or pos.total == 0 then
    return ""
  end
  return string.format(" [%d/%d]", pos.position, pos.total)
end

function M.setup()
  -- Register with lualine or statusbar if available (optional enhancement)
  -- This is optional — get_status() is the core API
end

return M
```

Key design decisions:
- `get_status()` is a standalone function — it pcall-requires navigation module to avoid circular deps
- Returns empty string when navigation is not active (so status line doesn't show stale data)
- Format: `[3/20]` (bracket-position-forwardslash-total-bracket) — follows diffview's existing style
- No UI rendering — just provides status text string
- Does NOT register any keymaps or autocmds
- If navigation module not available (Plan 01 not loaded), returns "" gracefully

Follow AGENTS.md patterns: `local M = {}`, function definitions, `return M` at end, double quotes, 2-space indent.
